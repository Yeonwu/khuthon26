# DigGak Frontend

이 문서는 DigGak 웹 프론트엔드의 주요 기능 요소 및 동작 방식을 요약한 가이드입니다. 

## 📌 주요 기능 (Features)

### 1. 오디오 파일 업로드 (Audio Upload)
- **Drag & Drop 지원**: 사용자가 직관적으로 오디오 파일을 끌어다 놓아 업로드할 수 있습니다.
- **포맷 지원**: `.mp3`, `.wav`, `.m4a`, `.aac`, `.ogg`, `.flac`, `.mp4` 등 다양한 오디오 포맷을 업로드할 수 있습니다.
- **진행 상태 표시**: 파일 선택 시 파일명과 용량이 표시되며, 업로드 진행 중에는 버튼 상태가 비활성화되어 중복 업로드를 방지합니다.

### 2. 실시간 작업 큐(Queue) 및 폴링 (Real-time Status Polling)
- **주기적 동기화**: `setInterval`을 이용해 3초마다 백엔드 API(`/api/v1/uploads`)를 폴링(Polling)하여 작업 리스트와 처리 상태를 최신화합니다.
- **상태 관리**: 업로드된 파일이 백엔드에서 처리되는 동안(Processing) 로딩 애니메이션과 함께 대기 상태를 나타내고, 처리가 완료되면 결과물 렌더링으로 전환합니다.

### 3. 결과물 오디오 재생 (Playback Surface)
- **인앱 오디오 플레이어**: 처리가 완료되어 생성된 오디오 결과물을 브라우저 내에서 즉시 들어볼 수 있는 재생/일시정지 기능을 지원합니다.
- **진행률 표시**: 오디오 재생 시간에 맞춰 진행 바(Progress bar)가 채워지며 현재 재생 위치를 시각적으로 제공합니다.

### 4. 파일 다운로드 (File Download)
- **결과물 저장**: 완료된 생성물 내역 우측의 `download` 버튼을 클릭하여 결과 오디오 파일을 클라이언트 로컬(PC, 모바일 기기)로 즉시 다운로드할 수 있습니다.
- 브라우저의 기본 파일 다운로드 동작(`window.URL.createObjectURL` 활용)을 통해 백그라운드 다운로드를 수행합니다.

### 5. 에러 핸들링 (Error Handling)
- 업로드 실패, 네트워크 오류, 서버 처리 오류 발생 시 화면에 빨간색 에러 메시지 박스로 오류 내용을 명확히 표시하여 사용자에게 안내합니다.

## ⚙️ 연동 정보 (API)
프론트엔드는 다음의 REST API 엔드포인트와 통신하여 동작합니다. (서버 주소는 환경 변수 `VITE_API_URL` 에 의해 결정됩니다.)
- `POST /api/v1/upload` : 폼 데이터(Form-data) 형식으로 오디오 파일을 업로드하여 분석/생성 작업을 요청합니다.
- `GET /api/v1/uploads` : 현재 등록된 작업 큐 상태와 완료된 결과물의 정보를 가져옵니다.
